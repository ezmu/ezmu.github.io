⚙️ Complete Configuration File

Here's the full configuration file with ALL available options:

# ============================================
# MIDKNIGHT CACHE PROXY CONFIGURATION
# ============================================

# ---------- NETWORK SETTINGS ----------
port = 8080                          # HTTP proxy port
backend = 127.0.0.1:3000             # Upstream backend server

# ---------- CACHE SETTINGS ----------
shm_size_mb = 512                    # Shared memory size (RAM cache)
max_entries = 10000                  # Maximum cache entries
cache_mode = config                  # all | config | auto
eviction_policy = LRU                # LRU | LFU | FIFO

# ---------- LICENSE ----------
license = MIDK-1234-5678-9012-3456   # Your license key (MIDK-XXXX-XXXX-XXXX-XXXX)

# ---------- DISK CACHE ----------
disk_cache_dir = /tmp/ezmu_cache     # Disk cache directory
disk_cache_size_mb = 10240           # Max disk cache size (10GB)

# ---------- ADMIN SETTINGS ----------
admin_allowed_ips = 127.0.0.1,192.168.1.0/24   # Allowed admin IPs
admin_secret = mysecretkey                     # Admin authentication

# ---------- PERFORMANCE ----------
max_threads = 0                      # 0 = auto (based on CPU cores)
enable_stats = 1                     # Enable statistics collection
enable_logging = 1                   # Enable request logging

# ---------- CACHE RULES ----------
# Format: cache_rule = pattern ttl storage
# ttl in seconds, storage: shm or disk

# Static assets - Long TTL, Disk storage
cache_rule = *.jpg 86400 disk
cache_rule = *.jpeg 86400 disk
cache_rule = *.png 86400 disk
cache_rule = *.gif 86400 disk
cache_rule = *.svg 86400 disk
cache_rule = *.webp 86400 disk
cache_rule = *.mp4 604800 disk
cache_rule = *.webm 604800 disk
cache_rule = *.pdf 604800 disk

# Code files - Long TTL, SHM storage (faster)
cache_rule = *.css 604800 shm
cache_rule = *.js 604800 shm
cache_rule = *.mjs 604800 shm

# Fonts - Very long TTL
cache_rule = *.woff 2592000 shm
cache_rule = *.woff2 2592000 shm
cache_rule = *.ttf 2592000 shm

# API endpoints - Short TTL
cache_rule = /api/* 3600 shm
cache_rule = /v1/* 7200 shm
cache_rule = /graphql 300 shm

# Search endpoints - SHM with short TTL
cache_rule = /search/* 600 shm
cache_rule = /products/* 1800 shm

# Admin section - NO CACHE
cache_rule = /admin/* 0 shm
cache_rule = /login 0 shm
cache_rule = /logout 0 shm

# User-specific - NO CACHE
cache_rule = /user/* 0 shm
cache_rule = /cart/* 0 shm
cache_rule = /checkout/* 0 shm

# HTML pages - Medium TTL
cache_rule = *.html 3600 disk
cache_rule = *.htm 3600 disk

# XML/RSS feeds
cache_rule = *.xml 1800 shm
cache_rule = *.rss 1800 shm

# Everything else - Default rule
cache_rule = * 1800 shm
💡 Note: Rules are evaluated in order. First matching rule applies!

🎯 Cache Rules Explained

Rule Syntax

cache_rule = <pattern> <ttl> <storage>

Pattern Matching

PatternMatchesExample
*.extAll files with extension*.jpg
/path/*All paths under directory/api/*
exact/pathExact path match/index.html
*Everything (default)*

TTL Values (Time To Live)

DurationSecondsUse Case
0No cacheDynamic/private content
5 minutes300Real-time data
1 hour3600API responses
1 day86400Images, static content
1 week604800CSS, JS libraries
1 month2592000Fonts, rarely changed assets
1 year31536000Permanent assets

💾 Storage Types Comparison

FeatureSHM (Shared Memory)DISK
Speed⚡ Nanoseconds (RAM)🐢 Milliseconds (Disk I/O)
CapacityLimited (set by shm_size_mb)Large (disk_cache_size_mb)
PersistenceLost on restartPersistent across restarts
Best forSmall files, APIs, JSONImages, videos, large files
CostUses RAMUses disk space
⚠️ Important: SHM cache is faster but limited. Use DISK for large files!

🔌 Complete Admin API Reference

Endpoints

MethodEndpointDescription
GET/__ezmu_admin__/statsCache statistics
GET/__ezmu_admin__/licenseLicense information
DELETE/__ezmu_admin__/cache/allClear all cache
DELETE/__ezmu_admin__/cache?key=KEYDelete specific entry

Examples

# Get statistics
curl http://localhost:8082/__ezmu_admin__/stats

# Clear specific cache
curl -X DELETE "http://localhost:8082/__ezmu_admin__/cache?key=/api/users"

# Reload config after editing midknight.conf
curl -X POST http://localhost:8082/__ezmu_admin__/reload

# Push data to searchable cache
curl -X POST http://localhost:8082/__ezmu_admin__/cache/json \
  -H "Content-Type: application/json" \
  -d '{
    "key": "/api/products",
    "data": "[{\"id\":1,\"name\":\"Laptop\",\"price\":999}]",
    "searchable_fields": "name,category",
    "sortable_fields": "price"
  }'

🔍 Searchable Cache Feature

What is Searchable Cache?

MidKnight can index JSON data and provide full-text search directly from cache!

Storing Searchable Data

# Via Admin API
curl -X POST http://localhost:8082/__ezmu_admin__/cache/json \
  -H "Content-Type: application/json" \
  -d '{
    "key": "/api/products",
    "data": "[
      {\"id\":1,\"name\":\"Gaming Laptop\",\"category\":\"electronics\",\"price\":1299},
      {\"id\":2,\"name\":\"Office Laptop\",\"category\":\"electronics\",\"price\":799},
      {\"id\":3,\"name\":\"Wireless Mouse\",\"category\":\"accessories\",\"price\":45}
    ]",
    "searchable_fields": "name,category",
    "sortable_fields": "price,name"
  }'

Search Queries

# Full-text search
curl "http://localhost:8080/api/products?q=laptop"

# Field search
curl "http://localhost:8080/api/products?category=electronics"

# Filter by price range
curl "http://localhost:8080/api/products?price__gt=500&price__lt=1000"

# Sort results
curl "http://localhost:8080/api/products?sort=price&order=desc"

# Pagination
curl "http://localhost:8080/api/products?page=2&per_page=20"

# Combine everything
curl "http://localhost:8080/api/products?category=electronics&sort=price&order=asc&page=1&per_page=10"

🔧 Troubleshooting Guide

Common Issues and Solutions

Issue 1: "Failed to initialize shared memory" 
# Solution: Increase system shared memory limits
sudo sysctl -w kernel.shmmax=1073741824
sudo sysctl -w kernel.shmall=2097152

# Make permanent
echo "kernel.shmmax=1073741824" | sudo tee -a /etc/sysctl.conf
echo "kernel.shmall=2097152" | sudo tee -a /etc/sysctl.conf
Issue 2: Cache not working 
# Check:
# 1. Is method GET?
# 2. Does path match any rule?
# 3. Is TTL > 0?
# 4. Is cache_mode correct?

# Debug: Check if rule matches
curl -I http://localhost:8080/your/path
Issue 3: High memory usage 
# Solution: Reduce SHM size or max_entries
shm_size_mb = 256
max_entries = 5000

# Or monitor and clear cache
curl -X DELETE http://localhost:8082/__ezmu_admin__/cache/all

🚀 Running the Server

Basic Commands

# Start with config
./midknight --config midknight.conf

# Start in background
./midknight --config midknight.conf > midknight.log 2>&1 &

# Start with command line overrides
./midknight --port 9090 --backend localhost:3000 --shm-size 1024

# Stop
pkill midknight

# Check if running
ps aux | grep midknight
netstat -tlnp | grep midknight

Systemd Service (Production)

# /etc/systemd/system/midknight.service
[Unit]
Description=MidKnight Cache Proxy
After=network.target

[Service]
Type=simple
User=www-data
ExecStart=/usr/local/bin/midknight --config /etc/midknight/midknight.conf
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

# Enable and start
sudo systemctl daemon-reload
sudo systemctl enable midknight
sudo systemctl start midknight
sudo systemctl status midknight
