Getting Started

Run MidKnight in 30 seconds — no dependencies, no configuration hell.

MidKnight is a single binary HTTP reverse proxy with built-in zero-copy cache, full‑text search, fuzzy search, and auto‑complete. It runs on any Linux system with zero external dependencies.

License

MidKnight requires a valid license key to run. The license is embedded in the binary and can be activated via configuration or environment variable.

# In config file
license = MIDK-XXXX-XXXX-XXXX-XXXX

# Or via environment variable
export MIDKNIGHT_LICENSE=MIDK-XXXX-XXXX-XXXX-XXXX

Installation

# Download the latest release
wget https:// mk domain /releases/midknight-latest-linux-amd64 -O midknight

# Make it executable
chmod +x midknight

midknight is a single file. No npm install, no apt-get, no docker pull. Just the binary.

Configuration

Everything is configured in a single file. No complex YAML, no environment variables, no secret management overhead.

# midknight.conf
port = 8081
backend = 127.0.0.1:3000
shm_size_mb = 512
license = MIDK-XXXX-XXXX-XXXX-XXXX

cache_rule = *.jpg 86400 disk
cache_rule = /api/* 3600 shm

ft_enabled = 1
ft_size_mb = 256

fuzzy_enabled = 1
ac_enabled = 1

Running MidKnight

# Start MidKnight
./midknight --config midknight.conf --port 8081 --backend 127.0.0.1:3000

# Test the proxy
curl http://localhost:8081/api/test

# Check cache stats
curl http://localhost:8082/__ezmu_admin__/stats

Create Index

Before loading data, you must create an index. Each index has its own isolated memory space.

# Create a products index
curl -X POST http://localhost:8082/__ezmu_admin__/ft/index/create \
  -H "Content-Type: application/json" \
  -d '{
    "name": "products",
    "language": "en",
    "size_mb": 128
  }'

Note Always create the index before loading data. The index name must match the one used in load_json.

List Indexes

curl http://localhost:8082/__ezmu_admin__/ft/index/list

Drop Index

curl -X DELETE "http://localhost:8082/__ezmu_admin__/ft/index/drop?index=products"

⚠️ This permanently deletes all data in the index.

Index Stats

curl "http://localhost:8082/__ezmu_admin__/ft/index/stats?index=products"

Load JSON

Load JSON data into an existing index. The data must be an array of objects with an id field.

# Sample JSON
{
  "data": [
    {"id": 1, "name": "Product A", "description": "..."},
    {"id": 2, "name": "Product B", "description": "..."}
  ]
}

# Load into index
curl -X POST http://localhost:8082/__ezmu_admin__/persistence/load_json \
  -H "Content-Type: application/json" \
  -d '{"index":"products","file":"/data/products.json"}'

Load from URL

curl -X POST http://localhost:8082/__ezmu_admin__/persistence/load_url \
  -H "Content-Type: application/json" \
  -d '{"index":"products","url":"https://example.com/products.json"}'

Batch Indexing

curl -X POST http://localhost:8082/__ezmu_admin__/ft/index/batch \
  -H "Content-Type: application/json" \
  -d '{
    "index": "products",
    "documents": [
      {"id": 1, "content": "Product A description"},
      {"id": 2, "content": "Product B description"}
    ]
  }'

Cache

Zero-copy shared memory (SHM) cache with disk fallback. Responses stored with TTL and evicted using LRU policy.

cache_rule = *.jpg 86400 disk
cache_rule = /api/* 3600 shm

Full-Text Search

Built-in full-text search with 8-language support. No Elasticsearch required.

curl "http://localhost:8081/api/search/products?q=water"

Fuzzy Search

Levenshtein-based fuzzy search with automatic suggestions.

curl "http://localhost:8081/api/search/countries?q=palestin"

Auto-Complete

curl "http://localhost:8081/api/ac?q=wa"

JSON Path

curl "http://localhost:8081/api/v1/path/countries/65/capital"
